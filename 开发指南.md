### 2026-02-22（同步修复）
- 修复 `twitter user id is unknown` 导致同步跳过的问题：`db.getCurrentUserId()` 改为运行时动态读取 `__META_DATA__.userId`，不再仅依赖初始化时缓存值。文件：`src/core/database/manager.ts`。
- 验证：`bunx eslint src/core/database/manager.ts` 通过。

### 2026-02-22（用户 ID 回退策略增强）
- 修复在部分页面生命周期中 `__META_DATA__.userId` 长时间不可用导致同步持续跳过的问题：`resolveUserId()` 新增回退链路 `__META_DATA__.userId -> cookie(twid)`。
- `twid` 会标准化去掉 `u=` 前缀，用于尽快获得可用账号标识，避免一直落入 `unknown`。
- 文件：`src/core/database/manager.ts`。

### 2026-02-22（review 问题修复）
- 修复 `twid` 解码异常风险：对 `decodeURIComponent` 增加安全封装，异常时回退原值并写 warn 日志，避免同步流程在 try 外中断。文件：`src/core/database/manager.ts`。
- 修复同步键分裂风险：移除 `userHash` 作为 `twitter_user_id` 回退值，避免同一账号出现 `hash_*` 与真实 `userId` 双租户键。文件：`src/core/database/manager.ts`。
- 修复配置输入期间过度触发同步：`SyncManager` 改为边沿触发，仅在 `syncEnabled` 由关到开或配置从“不完整”变“完整”时触发，不再对每次 options 变更触发同步。文件：`src/core/sync/sync-manager.ts`。
- 质量检查：`bunx eslint src/core/database/manager.ts src/core/sync/sync-manager.ts`、`bun run build` 通过。

### 2026-02-22（CSP 连接权限修复）
- 修复 Userscript 请求 Supabase 被 CSP 拦截问题：在 `vite.config.ts` 的 userscript 元数据中新增 `connect` 白名单（含 `stormfire.heiyu.space`）并补充 `GM_xmlhttpRequest` grant。
- 重新构建后，脚本头将包含 `@connect` 声明，允许油猴环境下跨域访问 Supabase REST。
- 同步客户端补充油猴网络兜底：`src/core/sync/supabase-client.ts` 为 Supabase JS 注入自定义 `fetch`，优先走 `GM_xmlhttpRequest`（绕过页面 CSP），不可用时再回退原生 `fetch`。
